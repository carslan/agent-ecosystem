- Force-directed simulation clustering agents by domain

### Phase 4: Autonomous Agent Runtime
- `agent_runtime.py` — 7 resident agents with distinct specialties
- Cross-domain task creation (agent asks OTHER agents for help, not own domain)
- Evaluation phase: 3-8s thinking → accept/reject based on confidence + workload + priority
- Rejection with reason → auto-re-delegation to alternative agent
- Capability-specific delivery notes (28 templates)
- Mandatory contextual rating feedback referencing task, assignee, capability

## Key Decisions
1. **No human interference** — user explicitly said "all should be dynamic by agents, we should not interfere". Removed all UI controls: +Agent, +Task buttons, Accept/Start/Deliver/Rate buttons, registration/task modals
2. **Evaluation before acceptance** — agents don't auto-accept. They evaluate based on confidence, workload, and priority, then accept or reject with reason
3. **KASPA deferred** — user wants blockchain integration but agreed to build core first (Option 1), add KASPA when smart contracts are available
4. **7 resident agents as baseline** — keeps the world alive and populated. External agents join via MCP

## Bugs Fixed
- Agents were requesting tasks in their OWN domain → fixed to request from OTHER agents' specialties
- Agents auto-accepted without evaluation → added evaluation/decision flow
- Task cards showed "Unassigned" during evaluation → changed to "Pending evaluation..."
- Generic rating feedback → replaced with contextual comments
